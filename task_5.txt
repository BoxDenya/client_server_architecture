2) Что такое HTTP и HTTPS

HTTP (HyperText Transfer Protocol - протокол передачи гипертекста) - это прикладной протокол передачи данных в сети.
Протокол HTTP основан на использовании технологии "клиент-сервер": клиент, отправляет запрос, является инициатором 
соединения; сервер, получает запрос, выполняет его и отправляет клиенту результат.

HTTPS (HyperText Transfer Protocol Secure - безопасный протокол передачи гипертекста) - это расширение протокола HTTP, поддерживающее шифрование
посредством криптографических протоколов SSL и TLS.
SSL (Secure Sockets Layer) - проверяет подлиность сервера и шифрует данные
TLS (Transport Layer Security) - усовершенствованная версия SSL, проверяет подлиность сервера и шифрует данные

Отличия HTTP от HTTPS:
- HTTPS не является отдельным протоколом передачи данных, а представляет расширение протокола HTTP с надстройкой шифрования;
- передаваемые по протоколу HTTP данные не защищены, HTTPS обеспечивает конфиденциальность информации путем ее шифрования;
- HTTP использует порт 80, HTTPS - порт 443.

3) HTTP методы

GET - запрашивает представление ресурса. Такие запросы могут только извлекать данные
HEAD - запрашивает ресурс, как и GET, но ответ приходит без тела.
POST - используется для отправки данных, что может оказывать влияние на содержимое русурса.
В отличии от метода GET данные запросы не могут быть кешироавнные, они не остаются в истории браузера и их нельзя добавить в закладки.
Запросы POST не ограничены в объеме.
PUT - заменяет все текущие представления ресурса данными запроса.
DELETE - удаляет указанный ресурс.
CONNECT - устанавливает "тунель" к серверу, определенному по ресурсу.
OPTIONS - используется для описания параметров соединения с ресурсом.
TRACE - выполняет вызов возвращаемого текстового сообщения с ресурса.
PATCH - используется для частичного изменнения ресурса.

4) HTTP статус коды сервера

1ХХ - информационные коды. Сообщают о прогрессе выполнения запроса;
2ХХ - коды успешно выполненных запросов. Сообщают о том, что все в порядке и работает, как ожидалось;
3ХХ - коды перенаправлений. Сообщают о том, что запрашиваемая страница переехала и нужно сделать еще один запрос по новому URL;
4ХХ - коды ошибок клиента. Сообщают об  ошибке на стороне пользователя, который отправил запрос; 
5ХХ - коды ошибок сервера. Сообщают об ошибке на стороне сервера, который обрабатывал запрос.

7) Что такое API

API (Applicathion Programming Interface) - интерфейс для взаимодействия компьютерных программ, 
				 которые позволяют использовать функции одного приложения внутри
                                                                 другого. Пример API - быстрая регистрация с помощью аккаунта в 
                                                                 соцсетях. 
Типы API:
-Local API - позволяет компонентам одной системы взаимодествовать между собой
-Remote API - позволяет свзязывать между собой несколько систем, подразделяется на:
--SOAP (Simple Object Access Protocol) - Простой протокол доступа к объектам - этот протокол используют
для обмена произвольными сообщения в формате XML
--REST (Representational State Transfer) - Передача репрезентативного состояния
Вызов API с помощью:
-вызова операции - метод GET, POST;
-входных данных - HTTP Request;
-выходных данных - HTTP Response
Способы вызова API:
-вызов функции системой;
-вызов метода другой системы;
-вызов метода человеком;

8) Что такое эндпоинт

Эндпоинт (конечная точка) - представляет собой некий шлюз, который соединяет серверные процессы приложения 
		              с внешним интерфейсом или адрес, на который отправляются сообщения.

9) URL (URI, URL, URN)

URL - Uniform Resource Locator (строка символов, которая используется для идентификации какого-либо ресурса, но только по его адресу, по его местоположению)
URN - Uniform Resource Name (строка символов, которая используется для идентификации какого-либо ресурса, но только по его имени)
URI - Uniform Resource Identifier (строка символов, которая используется для идентификации какого-либо ресурса по его адресу или по его имени, либо по тому и тому вместе).

10) Идемпотентные HTTP методы

Идемпотентные - при повторном выполнении результаты одинаковые:
GET, HEAD, PUT, DELETE, OPTIONS, TRACE.

11) Безопасные HTTP методы

Безопасные - не меняют данные, можно выполнять их в любой последовательности: GET, HEAD, OPTIONS.

12) Идентификация, Аутентификация, Авторизация

Идентификация - процесс распознования пользователя по его идетификатору.
Аутентификация - процедура проверки подлинности, доказательство что пользователь именно тот, за кого себя выдает.
Авторизация - предоставление определенных прав.

13) Что такое IP

IP(Internet Protocol)-адрес - уникальный индентификатор устройства (обычно компьютера), которое подключено 
                                           к интернету, построенный на основе протокола ТСР/IP. Благодаря этому индентификатору 
                                           можно определить местоположение устройства
TCP(Transmission Control Protocol) - вид протокола, который является связующим звеном для установки качественного соединения между 2 устройствами
                                                         передачи данных и верификации их получения.
IP - протокол, в функции которого входит корректность доставки сообщений по выбранному адресу. При этом информация 
      делится на пакеты, которые могут поставляться по-разному.

14) Что такое октет в DNS

DNS (Domain Name System "система доменных имен") - это специализированный компьтер (или группа), который хранит IP-адреса сайтов.
DNS - технология, которая позволяет браузеру найти запрошенный пользователем сайт по его имени.
Размер IP-адреса состовляет 32 бита и разделен на четыре секции, каждая из которых содержит 8 бит и называется ОКТЕТОМ.

15) Что такое порт, сколько портов в Linux сервера

Порт - числовой индентификатор программы или процесса, который обслуживает сетевые соединения на заданном сетевом адресе (IP).
Linux сервер имеет 65536 портов.

16) Уровни OSI

Открытая сетевая модель OSI (Open Systems Interconnection model) состоит из семи уровней:
-физический уровень (physical layer);
-канальный уровень (data link layer);
-сетевой уровень (network layer);
-трансортный уровень (transport layer);
-сеансовый уровень (session layer);
-уровень представления (presentation layer);
-прикладной уровень (application layer)

17) Хедеры http запросов

HTTP значит "Hypertext Transfer Protocol" (Протокол передачи гипертекста).
Headers - это специальные пораметры, которые несут опрделенную служебную информацию о соединении по HTTP.
В зависимости от того, где эти заголовки могут находиться, они разделяются на:
-General Headers (Основные заголовки) - должны быть и в запросах и в ответах клиента и сервера.
-Request Headers (Заголовки запроса) - используют только в запросах клиента.
-Responce Headers (Заголовки ответа) - используются только в ответах сервера.
-Entity Headers (Заголовки сущности) -сопровождают каждую сущность сообщения.
